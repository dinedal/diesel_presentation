APIService Framework
AKA Diesel / Distil / Dyslexic
What is it?
A DSL for request routing and parameter validation. Also: Racktivesupport web framework.
Features:
  • Sinatra-ish DSL
  • Completely describes valid requests
  • Generates API documentation
  • Super fast
What does it look like? 
get '/foo' do
    required :bar do
    type Array
    disallowed_values %w{zilch zip nada}
    end
end
What would this look like in Sinatra? 
get '/foo' do
    halt 404 unless params.has_key?(:foo)
    params[:foo] = params[:foo].split(' ')
    halt 404 if params[:foo].any? do |v|
        %w{zilch zip nada}.include?(v)
    end
end
You: C'mon guys, that's seriously bad enough that you had to reinvent the wheel?
There's a reason for doing this, other then a little clean up.

metrics, page_ids, post_ids

since, until
Dates (ISO-8601)
interval
List of allowed values.
And that conditions between parameters were simillar, but not identical...
Max size of the arrays varies on endpoint and query
[:since, :until] can also accept times, but only uses the timepart on FB Post Insights endpoints
How do you do all this validation in Sinatra?
Mix a messy module in.
And in this new, what did you call it, Racktivesupport, framework thingy?
Much cleaner, your validations are also your endpoints.
What it all means:
Code completely describes the API
Can generate docs of every route.

Parameter based routing
